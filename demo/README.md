**Действие:** Пользователь с уже заполненным профилем вводит `/profile`.

### 4. Изменить отдел пользователя
**Действие:** Пользователь с уже заполненным профилем вводит `/change_dept`.

**Логика бота:**
1. Проверяет роль пользователя (команда доступна только для роли `employee`)
2. Если проверка пройдена → показывает интерактивные кнопки с доступными отделами
3. Пользователь выбирает новый отдел
4. Бот обновляет запись в БД, сохраняет новый отдел
5. Показывает подтверждение с названием нового отдела

## Admin


### 1. `/start` — Онбординг или приветствие
**Когда использовать:** Первый запуск бота или повторный вход.

**Что происходит:**
- Бот проверяет, есть ли ваш профиль в базе данных
- **Если профиль заполнен** (роль=админ):
    - Показывает персонализированное приветствие «С возвращением, {Имя}!»
    - Подсказывает про команду `/info`
- **Если профиль пустой или новый пользователь**:
    - Показывает кнопки выбора роли: `👤 Сотрудник` / `🔑 Администратор`
